# UE C++ FPS JSON Box Spawner

This project is a small FPS gameplay prototype built in Unreal Engine using C++.  
The main goal is to fetch box data from an online JSON file and spawn interactive boxes in the world.  
Each box has its own color, health, and score value. The player can shoot the boxes, reduce their health, and gain score when they are destroyed.

---

# Project Logic

The project works in several steps.

### 1. Fetch JSON from Internet
The game sends an HTTP request to an online JSON endpoint.

The JSON contains:

- Box types, Color for each type, Health value, Score value, Transform data for each box

The response is parsed using Unreal's built-in JSON system.

---

### 2. Convert JSON to C++ Structs

The JSON data is mapped into C++ structs.

Example structs used in the project:

- `FBoxTypeData`, `FBoxTransformData`, `FBoxObjectData`

These store:

- box type name, color, health, score, transform data (location, rotation, scale)

---

### 3. Spawn Boxes

After parsing the JSON:

1. Each object entry is read
2. The system finds the matching **box type**
3. A box actor is spawned in the world
4. The transform from JSON is applied

Each box therefore appears exactly where defined in the JSON.

---

### 4. Dynamic Material Color

A material with a **color parameter** is used.

When the box spawns:

- A **dynamic material instance** is created and the color from JSON is applied in C++.

This allows every box to have a different color.

---

### 5. Shooting and Damage

The FPS player can shoot boxes.

When a bullet hits a box:

- Box health decreases by **1**, When health reaches **0**, the box is destroyed
- The player receives the **score value** defined in JSON

---

### 6. Score System

The player score is displayed using a UI widget.

The UI base class is created in **C++**, and the layout is extended using Blueprint.

Whenever a box is destroyed:

- The score is updated, and the UI refreshes to show the new value

# How to Test the Feature

1. Launch the project
2. Start the game (Play in Editor)
3. Boxes should spawn automatically from the JSON data
4. Each box should:
   - Have a different color
   - Have health
5. Shoot the boxes using the FPS weapon
6. Observe:
   - Health decreases when hit
   - Box disappears(destroy) when health reaches zero
   - Player score increases
   - UI updates with the new score

# Challenges Faced

Some parts required extra attention during development:

### JSON Parsing
Handling nested JSON structures and mapping them correctly to C++ structs required careful parsing.

### Error Handling
HTTP requests can fail, and JSON parsing can break if data is incorrect.  
Basic checks were added to prevent crashes and handle invalid responses.

### Keeping Everything in C++
The assignment required almost all logic to stay inside **native Unreal C++**, which meant avoiding Blueprint logic except for UI layout.
